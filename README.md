# Seamline Marker

Interactive marker-making module of the **Seamline** apparel software bundle.
One self-contained file — **double-click `SeamlineMarker.html`** to run it in any browser. No install, no server; work is autosaved in the browser between sessions.

## V2 comparison workspace

- **Document tabs** keep independent markers, Cut Plans, edits, undo/redo history, zoom/pan state, and layer settings. Open a marker in the active tab or as a new editable tab; dirty tabs are marked and require confirmation before closing.
- **Comparison layers** import another marker into the active field as a read-only overlay. The Layers dock controls visibility, opacity, X/Y position, origin alignment, side-by-side view, and difference highlighting. Active and comparison layers share the same zoom/pan camera while the active marker remains fully editable.
- Marker JSON v2 files can retain the comparison-layer configuration while remaining backward-compatible with v1 marker files.

## Interface

A **two-colour, Claude-style** shell — a warm neutral scale (surfaces, text, borders) plus one clay accent, with different segments told apart by neutral shades rather than extra hues. **Light and dark modes** (toggle in the header, follows your OS by default, remembered per browser). Pattern pieces are shades of the clay accent, keyed to size, each keeping its size label.

- **Marker-info ribbon** — docked at the **bottom of the screen**, AccuMark-style: theme toggle, editable marker name, an **order chip** (compact peek at model + size ratio; click to open the Order dock), and a hero readout: **editable fabric width** (change it to re-lay the marker at a different width — pieces return to the tray for a clean re-nest), **marker length** in metres with the marker's **garment count** beneath, **utilization**, and **consumption** = `(marker length + end allowance) ÷ garments` in metres/garment with an **editable end-allowance** field (default 4 cm).
- **Toolbar** — a Gerber-style ribbon: **Cut Plan / Marker** workspace toggle, then **File · Piece · Marker · Fabric · Advanced · View** tabs mirroring the classic Easy Marking layout (Advanced is reserved for split piece / placement templates / cut sequencing). New Order, Save, undo/redo, Auto-Nest and Help stay always visible. Zoom is canvas-anchored.
  - **File** — open / plot SVG / PNG / **Copy 1:1** (clipboard PNG at true scale for 96-dpi paste targets). **Piece** — rotate, flip, butt-slide, **fit piece** (one-piece auto-place), **duplicate placed**, return / **unplace small** / return all, plus the group tools: **bundle select / flip bundle**, **align L/R/T/B**, **marry / unmarry**. **Marker** — effort + compact, **flip marker X / Y / XY**, **measure tools** (point↔point, piece↔piece closest gap, piece↔edges), **approval flags** (Needs approval / Approved, saved with the marker). **Fabric** — rotation rule, edit order, **defect zones & splice lines** (see below). **Advanced** — **cut sequencing & cut data** (see below). **View** — **zoom window** (W), **thumbnail minimap** (click to jump the view), **restore points** (manual snapshots + one stored automatically before every Auto-Nest).
- **Multi-select & bundles** — Shift-click adds pieces to the selection, Shift-drag marquee-selects, Alt-click selects a garment's whole bundle. A multi-selection drags as one rigid group (validity checked against everything outside it), butt-slides together (L/U), returns together (Del), aligns to its own left/right/top/bottom edge, and can be **married** so the pieces select and move as one from then on. **Flip bundle** mirrors the group in place and safely reverts if there's no room. Each piece can also carry its own **buffer override** or be **blocked** (whole bounding box becomes the obstacle, like Gerber's Block) from the Piece dock.
- **Fabric handling (Fabric tab)** — for spreading real rolls. **Defect zones**: drag a rectangle over a flaw (needle line, hole, dye streak) and pieces keep clear of it, in placement *and* auto-nest; select to move/resize with 8 handles, Delete to remove. **Splice lines**: mark where a fabric roll joins during spreading — no piece may straddle a splice (it would be cut across a seam in the ply); click to add, drag to move, or **Auto splice** every roll length (each join nudged to the nearest clear gap). Both are enforced from the same collision gate as piece overlap, so the nester respects them for free. A **retroactive check** flags any already-placed piece that a newly dropped/dragged guide now violates (red dashed outline) and **blocks cut-data export until resolved**, with a one-click **Nudge offenders** to shift straddlers clear. Guides live in the marker file, ride the same undo history as piece edits, and are excluded from the cut geometry. *(Deferred, data model pre-provisioned: overlap/OL allowance + fold pairing, spread-type engine, plaid/stripe matching, block fuse — the last two are woven-specific and low-value for a knit shop.)*
- **Cut sequencing & cut data (Advanced tab)** — **Auto sequence** orders the placed pieces the way a windowed conveyor cutter consumes a marker: bands by leading edge (band width = the editable "cut window", default 50 cm), serpentine within each band, near-ties cut the smaller piece first while the lay is still anchored. **Pick order** lets the operator click pieces in cut order on the canvas (click a numbered piece to un-pick it and everything after; Enter finishes, unpicked pieces follow in their previous order); **Reverse** flips the sequence. The **Knife path** overlay shows each contour re-stroked from its plunge point (a convex corner nearest the previous knife-up position — corner plunges hide the entry nick), the travel direction, the numbered order, and the dry-haul as a red dashed line, with cut length / dry-haul % in the status bar. **Generate Cut Data** exports **HPGL** (.plt, 1 plu = 0.025 mm, absolute PU/PD, command order = cut order) or **DXF R12** (AAMA-style: layer 1 = closed polylines in cut order, layer 15 = labels with the sequence number leading; correct extents, no R13+ entities). The sequence is saved in the marker file; re-nesting asks before discarding a manual sequence, and pieces added later are appended — never silently left uncut. Cut direction (CCW/CW) is normalized per piece, mirrored instances included.
- **Fabric categories** — each piece takes a free-text fabric code in the order form (A/B/C, 1/2/3, or BD/Gst/FR-style — up to 4 characters, default `A`); pieces sharing a code cut from the same fabric. When a marker uses more than one fabric, **chips above the tray** filter the tray to one fabric and **scope Auto-Nest to it**, so each fabric's marker can be laid one at a time. The Piece dock, tray rows, tooltips and the Order panel all show the fabric.
- **Coordinates** — the marker origin **0,0 is the bottom-left corner** (width ruler counts up from the bottom selvage; the cursor readout and piece positions follow the same convention).
- **Tray** — a scrollable left dock grouped by piece × size × fabric with count badges and a filter field.
- **Right dock** — four tabs:
  - **Piece** — the organised "change this piece" toolbar (mirrors AccuMark's Piece tab + Toolbox): Rotate 90° cw/ccw (exact angle for free-lay pieces), Flip H/V, Butt/slide in four directions, Align (centre in width / top / bottom edge), Return to tray — plus the selected piece's identity, dimensions and position. Surfaces automatically when you select a piece.
  - **Order** — read-only per-line breakdown; editing is the separate explicit "Edit order…" action.
  - **Options** — everything that used to be scattered: theme, grid / snap / auto-butt, rotation rule, cutter buffer, fabric width, end allowance, and a size-colour legend.
  - **Keys** — the keyboard reference.
- **Status bar** — persistent and colour-coded (info / selection / ok / warn / error); a warn/error message holds until the next one instead of vanishing like a toast.
- **Canvas** — piece labels counter-scale against zoom so they stay legible on long markers, fading out (revealed on hover/selection) once a piece is too small on screen to matter.
- **Cut Plan** is a workspace mode, not a modal — demand form, results and the marker canvas share one screen.

## Compared to AccuMark Easy Marking

Modeled on the Easy Marking ribbon (File · Piece · Marker · Fabric · Advanced · View) and Toolbox. **Implemented:** interactive placement, auto-nest (AutoMark analog), per-piece rotate/flip/butt/slide/align/centre/unplace, multi-select & bundles (select/flip/return, group drag/slide, align, **marry**, per-piece buffer/block), measure tools, flip marker X/Y/XY, fit piece, duplicate placed, unplace small, thumbnail minimap, zoom window, restore points, approval flags, compact, marker length/utilization/consumption/status, rotation rules & buffers, size grading, plot (SVG/PNG/Copy 1:1) & marker JSON, **cut sequencing with knife-path preview and Cut Data generation (HPGL + DXF R12)**, **fabric handling (defect zones + splice lines, with retroactive validation and an export guard)**, and the Cut Plan (Easy Plan analog). **Not yet implemented** (candidates for next passes): overlap/OL allowance + fold pairing, spread-type engine, plaid/stripe matching and block fuse (woven-specific), bump lines, split piece, dynamic alteration, stored layrules (placement templates), marker copy/attach/split, and the marker library / ERP import (cloud phase).

## Workflow (modeled on the classic marker pipeline)

1. **New Order** — an Easy Order-style 5-step form (Models → Sizes → Pieces → Fabric → Finish): add **one or more models** (T-Shirt, Shirt, Trousers, Bra, Panty) as order lines, set size quantities per model, untick pieces to exclude them, **set each piece's lay direction** (two-way by default, overridable to four-way or free per piece), choose fabric width / cutter buffer / lay rule, review the summary with an estimated marker length, then **Create** or **Create + Auto-Nest**. All models nest together in one combined marker; pieces carry a model tag (TSH·, SHT·, TRS·) when the order is multi-model.
2. **Place** — click a tray piece then click the fabric (green ghost = valid, red = blocked), or press **⚡ Auto-Nest** to pack the whole marker automatically (effort slider trades speed for tightness).
3. **Refine** — drag pieces (overlaps bounce back), rotate/flip under lay-rule constraints, **Butt** slides a piece until it touches its neighbour, **Compact** tightens the whole marker. The header updates live: editable **fabric width**, **marker length** in metres (3 dp) with the marker's **garment count**, **utilization %**, and **consumption** — fabric per garment, `(marker length + end allowance) ÷ garments`, in metres/garment (3 dp), with an editable **end allowance** (default 4 cm) — plus piece counts and the MADE / PARTIAL / UNMADE status chip.
4. **Output** — **Save** the marker as JSON (re-open later with **Open**), export the plot as **SVG** (1 unit = 1 cm, with title block) or **PNG**.
5. **Cut Plan** (Easy Plan analog) — for bulk production: enter demand per **colour × size** plus ply limits (min/max ply, max garments per marker), hit **Calculate markers**, and the demand is split into a marker set with spreads (colour × plies) — proportional demand collapses into one shared ratio marker; residuals are cleared exactly by per-colour cleanup markers, with below-min-ply spreads flagged. **Auto-nest all markers** draws every layout in the plan and replaces the length estimates with actual nested lengths and utilizations. **Mini-markers PDF** downloads the whole plan as a dependency-free PDF — a summary page (demand, settings, marker table) plus one page per marker with the nested layout drawing, ratio, spreads, plies, garments, length, utilization and fabric usage. Open any plan row as a marker order, or export the cut tickets as CSV.

## Rules engine

- Pieces can never overlap; an optional buffer (0.3 / 0.5 / 1 cm) keeps a minimum cutter gap between pieces.
- Every piece carries a lay direction — body panels default to two-way (0°/180°), small parts four-way, bindings free — set per piece in the order form, with a global rotation rule (strict / 90° ok / free) that can further relax (never tighten) all pieces at once for non-directional fabric. (Renamed from "lay rule" to avoid clashing with Gerber's Layrules, which are saved placement templates — a planned feature.)
- Paired pieces (shirt fronts, trouser legs) are cut mirrored automatically.
- **Size scales** are per model, so garments on different sizing systems can share one marker:
  - `alpha` — XS–XXL (T-Shirt, Shirt, Trousers, Panty). Length grades at half the girth rate.
  - `bra` — **32A–42D** (24 sizes: bands 32–42 × cups A–D). Two-dimensional grading: each piece declares what drives it, so band size grows the wings, under band and straps, while the cup letter grows the cup panels — a 36D has the same 68 cm band as a 36B but a larger cup. Size chips are coloured by hue = cup, shade = band.
- Adding a model is data-only: give it a piece list and (optionally) a `scale`; the order form, cut plan, marker labels and PDF pick it up automatically.

## Keyboard

`R`/`Shift-R` rotate · `F`/`Shift-F` flip · arrows nudge 1 cm (`Shift` = 1 mm) · `L`/`U` butt-slide · `Del` return to tray · `Esc` cancel · `Ctrl-Z`/`Ctrl-Y` undo/redo · `+`/`−`/`0` zoom · drag background to pan · `N` new order · `?` help
